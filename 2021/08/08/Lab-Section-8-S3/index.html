[Lab] Section 8: S3

Cùng thực hành một số bài Labs để hiểu hơn về S3 trên AWS nhé!

Lab 01: S3 Hands On

Bài thực hành sẽ hướngd dẫn bạn các thao tác cơ bản với S3 như tạo bucket, upload các objects!

Bước 1: Truy cập dịch vụ S3 => Create bucket

The bucket name have to be globally unique

  • Bucket name: s3-demo-0808

Next, we have to select a region, so the region for our bucket, it could be some thing very close to you,

Dịch vụ S3 là global nhưng các S3 cần được tạo và đặt ở 1 region cụ thể, hãy nhớ điều đó

  • Region: EU
  • Bucket settings for Block Public Access: Tạm để Block all public access
  • Bucket Versioning: Disable
  • Default encryption: Disable
  • Click Create bucket

Bước 2: Upload Object lên S3

Trong tab Objects => Upload => Add files/Add folder. Sau đó chọn file mà bạn muốn upload lên. Kết quả sẽ như vậy

Bước 3: Làm sao để mở file và xem nội dung

Nhấn vào object để xem các thông số chi tiết

Hãy mở object theo 2 cách

  • Cách 1: Nhấn Object Actions (góc trên bên phải) => Open

Bạn sẽ nhìn thấy bức ảnh. Như bạn đã thấy, URK rất rất dàu và URL này được gọi là pre-signed URL và nó chưa AWS credentials trên URL. Mặc dù nó không public nhưng nó đang nói rằng “Hey, đây là credentials của tôi, hãy cho tôi xem bức ảnh đó”

  • Cách 2: Nhấn vào Object URL

Bạn sẽ không thấy bức ảnh, lỗi AccessDenied do các policy config đã chặn public.

Bước 4: Tạo folder

Từ bucket => Create folder

  • Folder name: images
  • Server-side encryption: Disable
  • Click Create folder

Tương tự, bạn hãy upload file vào thư mục này Add files. Nhìn lại thư mục thì có vẻ nó là các file, thư mục như Dropbox hay Google Drive, nhưng tất cả chúng là objects trong nội bộ S3

Lab 02: S3 Security: Bucket Policy Hands On

Bài thực hành sẽ làm hiển thị file ảnh (coffee.jpg) ở Lab 01 mà không cần pre-signed URL.

Chọn bucker => Permission Tab

To write a public bucket policy, I must first disable all the public access settings that are turned on for these buckets.

Do đó bạn nhìn phần Block public access (bucket settings) => Click Edit => Bỏ Select Block all public access => Click Save Changes

Đến phần Bucket policy => Edit

Bạn thấy có 2 tuỳ chọn là Policy examples hoặc Policy generate. Trong bài thực hành sẽ chọn Policy generate

Step 1: Select Policy Type

  • Select Type of Policy: S3 Bucket Policy

Step 2: Add Statement(s)

  • Effect: Allow
  • Principal: *
  • Actions: GetObject
  • Amazon Resource Name (ARN): copy ARN của S3 và cần thêm /* để lấy toàn bộ Objects

Ví dụ: arn:aws:s3::abc/*

  • Click Add Statements

Step 3: Generate Bucket Policy

Click Generate Policy. Bạn sẽ được Policy như mong muốn

Quay trở lại bước Bucket Policy => Paste giá trị Policy này vào. Click Save Changes

Lúc này, hãy để ý Access là Public, khi đó bạn mới có thể xem ảnh

Click lại URL trong object đó, bạn đã thấy show được ảnh

Lab 03: S3 Website Hands On

Đầu tiên bạn hãy upload 2 ảnh lên s3 bucket như vậy (Các bucket Policy giữ nguyên như bài Lab trước đó)

Bước 1: Upload file index.html

Upload file index.html lên bucket

Bước 2: Cấu hình enable website

Vào Bucket => Properties Tab. Tìm đến phần Static website hosting. Click Edit

  • Static website hosting: Enable
  • Hosting type: Host a static website
  • Index document: index.html
  • Error document: error.html

Click Save changes

Bước 3: Kiểm tra

Kéo chuột xuống phần Static website hosting, bạn sẽ thấy endpoint

Click vào URL này và kết quả bạn sẽ được một trang static website =))

Như vậy việc triển khai một static website hosting thật là đơn giản trên S3. Bạn chỉ cần đảm bảo 2 việc

  • Có file ảnh và index.html
  • Có S3 bucket policy public (bỏ block public như bài Lab 02) để mọi người có thể truy cập file HTML và ảnh. Rất dễ dàng =))

Lab 04: S3 Versioning Hands On

Chúng ta sẽ thực hành sử dụng Versioning trong S3, để xem các thay đổi. Bài thực hành sẽ vẫn sử dụng S3 bucket của các bài thực hành trước

Bước 1: Enable

Vào Bucket => Properties Tab => Bucket Versioning => Edit

  • Bucket Versioning: Enable
  • Click Save Changes

Bước 2: Kiểm tra

Upload lại file index.html với content đã được thay đổi

Từ Bucket => Chọn file index.html => Chọn Versions Tab

Do chưa enable version trong lần upload file đầu nên version sẽ là null

Bước 3: Rollback

Quay trở lại Bucket => Objects Tab chọn List versions, bạn sẽ thấy các version trong bucket của bạn

Bây giờ chúng ta sẽ khôi phục file index.html về bản cũ. Click chọn index.html (current version) => Delete

Load lại static website host trong Lab 03 và bạn sẽ thấy nội dung file index.html cũ

Bước 4: Rollback file đã bị xoá

  • Từ bucket chọn file coffee.jpg, xoá file này. Lúc này bạn load lại static website từ s3 sẽ bị lỗi ảnh
  • Restore coffee.jpg file. Để làm việc này, bạn trở lại bucket => Enable List versions, thực tế bạn sẽ cảm ơn versioning vì nó đã không xoá object ở high level.
  • Chọn version hiện tại của file coffee.jpg => Delete như bước 3 để rollback, từ đó khôi phục được file. Refresh lại static website s3 để xem kết quả.

Lab 05: S3 Server Access Logging Hands On

Phần này chúng ta cùng thực hành để hiểu dùng S3 chứa file log

Bước 1: Tạo bucket mới

Tạo bucket mới để chứa các object logs

  • Bucker name: lab-server-access-logging-0808
  • Các tham số khác chưa cần chỉnh. Click Create bucket

Lúc này chúng ta đang có 2 bucket trên S3:

Bước 2: Config server access logging

Vào bucket bài Lab 01 => Properties Tab => Server access logging => Click Edit.

  • Server access logging: Enable
  • Target bucket: Chọn bucket đã tạo ở bước 1 và thêm /logs để nhận biết. Ví dụ s3://lab-server-access-logging-0808/logs
  • Click Save Changes

Bước 3: Kiểm tra

Truy cập lại static webstie s3 của bài Lab trước hoặc truy cập vào các file ảnh để ghi log

Bạn cần chờ một khoảng thời gian, bạn kiểm tra lại trong bucket lab-server-access-logging-0808. Nên nhớ rằng sẽ cần 1 khoảng thời gian để có thể ghi log chứ không ngay lập tức được

Now, it’s not immediate. You’re not going to refresh right now and see logs. It’s going to take a little bit of time, but let’s wait. Usually it takes an hour or two.

Sau đó bucket này sẽ chứa các file logs như sau:

Chọn 1 file => Actions => Download để xem nội dung chi tiết. Bạn sẽ thấy được địa chỉ IP, thời gian truy cập, bucket nào, truy cập tài nguyên nào trên bucket:

Lab 06: S3 Replication Hands On

Bài thực hành sẽ hướng dẫn bạn tạo bucket mới sao chép, “nhân rộng” S3 bucket được chỉ định sang một hay nhiều region khác.

Bước 1: Tạo bucket mới

  • Bucket name: lab-replication-0808
  • Region: Chọn Region nào đó các với region bạn chọn cho các bài lab trước. Ví dụ us-east-1. Bucket này sẽ là bản sao của region Japan
  • Bucket versioning: Enable

Để chức năng replication hoạt động, bạn cần enable Bucket Versioning ở cả 2 bucket (bucket sẽ replication và bucket chứa replication)

  • Click Create bucket

Lúc này chúng ta đã có 3 bucket. Cái đầu tiên là bucket chúng ta sẽ tạo bản sao, cái giữa là cái bucket chính và cái cuối cùng là chứa server logs đã được tạo từ Lab 04

Bước 2: Cấu hình tạo bản sao từ main bucket tới replication bucket

Vào main bucket => Management Tab => Replication rules => Create replication rule

  • Replication rule name: demorule
  • Status: Enabled
  • Choose a rule scope: This rule applies to all objects in the bucket

  • Destination

    • Choose a bucket in this account
    • Bucket name: Name bucket đã tạo bước 1: lab-replication-0808. Bạn sẽ thấy thông tin Destination Region là us-east-1 giống region của replication bucket

  • IAM: Create new role

  • Click Save

Bước 3: Kiểm tra

Các objects trước khi enable chức năng sao chép sẽ không được sao chép. Do đó để test chức năng này, bạn cần upload file mới

Upload file mới lên main bucket và load lại replication bucket. Kết quả replication bucket cũng chứa file đó. Bạn phải chờ tầm 5s để có file, quá trình sao chép cũng không real time được

Khi sao chép, versioning của 2 bucket cũng giống nhau

Đây là versioning của main bucket

Như vậy chúng ta đã tạo được bucket sao lưu giữa 2 region, region Japan và region us-east-1. Thật tuyệt vời.

Lab 07: S3 Storage Classes Hands On

Bước 1: Upload

Vào bucket ở Lab 01 => Upload. Bạn scroll xuống phần Additional upload options

  • Storage class: Bạn nhìn thấy toàn bộ Storage Class trong S3
    • Với các TH Upload thông thường, mặc định sẽ là Standard
    • Trong bài thực hành sẽ chọn Standard-IA (các loại data ít được truy cập)
  • Click Upload

Bước 2: Kiểm tra

  • Khi đã upload xong, chọn object đó
  • Scroll xuống phần Storage class, bạn sẽ thấy Stardard-IA
  • Để thay đổi Storage class => Click Edit

Và khi bạn thay đổi Storage class như vậy, versioning cũng sẽ ghi nhận, very cool

Bước 3: Lifecycle rule

Từ bucket => Chọn Management Tab => Lifecycle rules => Create lifecycle rule

  • Lifecycle rule name: DemoRule
  • This rule applies to all objects in the bucket
  • Enable: I acknowledge that …
  • Lifecycle rule actions: Có rất nhiều action cho bạn tuỳ chọn
    • Chọn Transition current versions of objects between storage classes
  • Storage class transitions: Standard-IA
  • Days after object creation: 35

Bạn cũng có thể thêm transition bằng cách Add transition. Ví dụ bạn thêm

  • Storage class transitions: Glacier
  • Days after object creation: 180

Bạn có thể tuỳ chọn thêm Expire current versions of objects trong Lifecycle rule actions để config thêm nếu muốn

And I say yes, sounds good. Next, I can also choose to expire current objects, or even delete them.
And so if I do so, I say, after 365 days, please expire an object. And this will add a delete marker.

Và cũng có thể tuỳ chọn Permanently delete previous versions objects

And you can play around, and you can delete permanently, previous objects or versions after say, 720 days, who knows, right?

Và bạn sẽ thấy một timeline về việc đổi storage class hay xoá preversion như bạn đã config ở trên. Một timeline cực kì trực quan ^_^

These Lifecycle Rules really allow you to optimize your cost, and they can be very handy if you have lots of objects in your S3 buckets

Trên đây là các bài thực hành cơ bản về S3, tôi hi vọng bạn thích nó và chúng ta sẽ gặp lại nhau trong các bài thực hành tiếp theo nhé <3
